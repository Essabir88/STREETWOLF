# الاستضافة المجانية — الطريقة الفعلية

## اقرأ هذا أولاً

**Vercel لا يصلح كما هو شائع.** [Sure] الخطة المجانية "Hobby" في Vercel تمنع
صراحة الاستخدام التجاري حسب شروط الاستخدام — أي مشروع يُدرّ دخلاً (متجرك هذا)
يجب أن يكون على خطة Pro المدفوعة (20$/شهر). هذا ليس قيداً تقنياً بل شرط
استخدام، وتجاهله يعرّض حسابك للإيقاف.

**قاعدة SQLite التي كانت في النسخة السابقة من هذا المشروع لم تكن لتعمل على أي
استضافة مجانية شائعة أصلاً.** الاستضافات المجانية (Vercel، Netlify، Render...)
تشغّل الكود على خوادم بدون قرص دائم — أي ملف تكتبه يُحذف عند كل إعادة تشغيل.
لهذا السبب رُحّل هذا المشروع بالكامل من SQLite إلى **PostgreSQL حقيقي** (راجع
`src/db/schema.ts` و`src/db/index.ts`) قبل كتابة هذا الدليل — بدون هذا
الترحيل، لا توجد طريقة مجانية تستضيف بها هذا الكود بشكل يحفظ بيانات الزبائن.

**قاعدة Postgres المجانية في Render تُحذف نهائياً بعد 30 يوماً** من الإنشاء —
بدون تحذير كافٍ، بدون نسخ احتياطي. لا تستخدمها لبيانات حقيقية.

## الخلاصة: المسار الموصى به (مجاني بالكامل، قانوني للاستخدام التجاري، لا يحذف بياناتك)

| الطبقة | الخدمة | لماذا |
|---|---|---|
| قاعدة البيانات | **Neon** (neon.tech) | Postgres حقيقي، مجاني دائماً بدون بطاقة بنكية، لا يحذف البيانات (يتوقف مؤقتاً بعد 5 دقائق خمول ويستيقظ خلال أقل من ثانية) |
| استضافة الموقع | **Netlify** (الأفضل) أو **Render** (الأبسط) | الاثنان يسمحان صراحة بالاستخدام التجاري على الخطة المجانية |

## الخطوة 1 — قاعدة البيانات على Neon

1. اذهب إلى neon.tech وسجّل حساباً (GitHub كافٍ، بدون بطاقة بنكية).
2. أنشئ مشروعاً جديداً. اختر منطقة قريبة (أوروبا مثلاً).
3. انسخ رابط الاتصال (connection string) الذي يبدأ بـ `postgres://...` —
   هذا هو `DATABASE_URL`.
4. محلياً: ضعه في `.env`، ثم:
   ```bash
   npm install
   npm run db:migrate
   npm run db:seed
   npm run dev
   ```

نفس رابط الاتصال هذا يصلح للتطوير المحلي وللإنتاج معاً — لا حاجة لتثبيت
Postgres على جهازك.

## الخطوة 2 — استضافة الموقع

### الخيار أ: Netlify — [Probable] الأنسب لتجربة زوار سلسة

Netlify يدعم Next.js كإطار عمل رسمي (بدون إعداد يدوي عادة)، ولا يوجد لديه
مشكلة "تبريد" (cold start) طويلة مثل Render.

1. ادفع الكود إلى مستودع GitHub.
2. netlify.com → "Add new site" → "Import an existing project" → اربط GitHub
   واختر المستودع.
3. يكتشف Netlify أنه مشروع Next.js تلقائياً. اترك إعدادات البناء الافتراضية.
4. أضف متغيرات البيئة من Site settings → Environment variables:
   `DATABASE_URL`، `JWT_SECRET` (قيمة عشوائية جديدة حقيقية —
   `openssl rand -base64 32`)، `POINTS_PER_MAD`، `MAD_CENTS_PER_POINT`،
   `NEXT_PUBLIC_SITE_URL` (رابط Netlify الذي سيُعطى لك، أو نطاقك الخاص).
5. Deploy. تحصل على رابط `something.netlify.app` خلال دقائق.

لأن Next.js 16 حديث نسبياً، إن واجهت أي خطأ غريب في البناء على Netlify تحديداً
(نادر لكن وارد مع أحدث إصدارات الأطر)، انتقل للخيار ب — نفس الكود يعمل عليه
بدون أي تعديل.

### الخيار ب: Render — [Sure] الأبسط والأكثر يقيناً، بتكلفة بطء أول زيارة

1. ادفع الكود إلى GitHub (كما أعلاه).
2. render.com → "New" → "Web Service" → اربط المستودع.
3. Environment: Node. Build command: `npm install && npm run build`.
   Start command: `npm start`.
4. أضف نفس متغيرات البيئة أعلاه.
5. Deploy.

**العيب الوحيد:** خدمة Render المجانية تتوقف بعد 15 دقيقة خمول، وتحتاج
30-60 ثانية لإعادة التشغيل عند أول زيارة تالية. مزعج لكن لا يفقد أي بيانات
(البيانات في Neon منفصلة تماماً وغير متأثرة).

## بعد أول نشر

- شغّل `npm run db:migrate` مرة واحدة على قاعدة Neon (من جهازك، بنفس
  `DATABASE_URL` المستخدم في الإنتاج) قبل أول زيارة حقيقية.
- اربط نطاقاً مخصصاً (مثلاً streetwolf.ma إن توفر، بعد حل تعارض الاسم في
  BRANDING.md) من إعدادات Netlify/Render مباشرة — كلاهما يدعم هذا مجاناً.
- لا تنسَ `JWT_SECRET` حقيقياً، ليس القيمة التجريبية.
