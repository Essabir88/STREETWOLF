# Street Wolf — متجر إلكتروني

متجر مبني بالكامل بكود مخصص: تيشورتات بقصة خاصة، إصدارات محدودة الكمية، ونظام
نقاط ولاء حقيقي (حسابات مستخدمين فعلية، لا مجرد عداد في المتصفح).

اقرأ أيضاً هذه الملفات قبل الإطلاق — كل واحد يوثّق قراراً مهماً اتُخذ أثناء
بناء هذا المشروع:

- **DEPLOYMENT.md** — كيف تستضيف الموقع مجاناً فعلياً (وأين لا تصلح Vercel/Render)
- **PAYMENTS.md** — لماذا Stripe غير مفعّل، وما البديل في المغرب
- **BRANDING.md** — تعارض اسم "Street Wolf" + عدم اتساق الشعار في ملفاتك

## الستاك التقني

- **Next.js 16** (App Router) + TypeScript — انتبه: `middleware.ts` أصبح
  `proxy.ts` في هذا الإصدار (`src/proxy.ts`)، وهو التغيير الذي حذّر منه
  `AGENTS.md`.
- **next-intl** لتعدد اللغات (FR/EN/AR، مع RTL كامل للعربية) — راجع القسم أدناه.
- **Drizzle ORM** + **PostgreSQL** (يعمل مع أي Postgres قياسي — Neon مُوصى به، مجاني، راجع DEPLOYMENT.md)
- مصادقة مبنية يدوياً: `bcryptjs` (تشفير كلمات السر) + `jose` (جلسات JWT في كوكيز httpOnly)
- **Tailwind CSS v4** (نظام تصميم مخصص عبر `@theme` في `globals.css`)
- خطوط مستضافة ذاتياً عبر `@fontsource` (لا تعتمد على اتصال بخوادم Google وقت البناء)
- **Vitest** لاختبار منطق الشراء (`src/lib/orders.ts`) ضد قاعدة Postgres حقيقية
  في الذاكرة (PGlite)، وليس مجرد mocks — راجع قسم الاختبارات أدناه.

`npx tsc --noEmit` و`npm run lint` نظيفان تماماً، و`npm test` (39 اختباراً)
ناجح بالكامل. **تنبيه مهم**: `npm run build` الكامل ضد Postgres حقيقي (Neon
أو أي مزوّد آخر) **لم يُتحقق منه بالكامل** في بيئة البناء هذه (لا Docker ولا
Postgres محلي متوفر هنا) — تحقّق منه بنفسك قبل النشر:
`DATABASE_URL=<رابطك> npm run build`.

## التشغيل محلياً

يحتاج المشروع الآن قاعدة PostgreSQL (لم يعد يستخدم ملف SQLite مرفق — راجع
DEPLOYMENT.md لماذا). أسرع طريقة: أنشئ قاعدة Neon مجانية (دقيقتان، بدون بطاقة
بنكية) واستخدم نفس الرابط محلياً وفي الإنتاج.

```bash
npm install
cp .env.example .env   # ضع رابط اتصال Postgres الحقيقي في DATABASE_URL
npm run db:migrate     # ينشئ الجداول
npm run db:seed        # يضيف 3 منتجات تجريبية (يتجاهلها إن كانت موجودة)
npm run dev
```

افتح http://localhost:3000. سجّل حساباً جديداً من `/register` لتجربة السلة،
الدفع عند الاستلام، وتراكم النقاط.

## أوامر مفيدة

| الأمر | الوظيفة |
|---|---|
| `npm run dev` | تشغيل محلي مع Turbopack |
| `npm run build` | بناء نسخة الإنتاج (يحتاج DATABASE_URL صالحاً لأن sitemap يقرأ المنتجات) |
| `npm run lint` | فحص الكود (نظيف حالياً، صفر أخطاء) |
| `npm test` | اختبارات Vitest — منطق الشراء ضد Postgres في الذاكرة، بما فيها اختبار تزامن المخزون |
| `npm run db:studio` | واجهة مرئية لتصفح قاعدة البيانات (Drizzle Studio) |
| `npm run db:generate` | توليد migration جديد بعد تعديل `src/db/schema.ts` |
| `npm run db:migrate` | تطبيق الـ migrations |
| `npm run db:push` | دفع الـ schema مباشرة بدون ملفات migration (أسرع للتجربة) |
| `npm run db:seed` | إضافة/تحديث المنتجات التجريبية |

## هيكل المشروع (الأهم فقط)

```
src/db/schema.ts               — الجداول: users, products (نصوص متعددة اللغات jsonb), orders, orderItems, pointsTransactions
src/db/seed.ts                 — 3 منتجات تجريبية بقصص مترجمة فعلياً FR/EN/AR (استبدلها بمحتواك)
src/lib/auth.ts                — تسجيل الدخول، الجلسات (bcryptjs + jose)
src/lib/orders.ts              — منطق الشراء كاملاً داخل transaction واحدة (المخزون + النقاط)
src/lib/points.ts              — معادلة كسب/استبدال النقاط والعملة (درهم مغربي، تنسيق حسب اللغة)
src/lib/payments.ts            — طرق الدفع المتاحة (COD فقط اليوم) — راجع PAYMENTS.md
src/lib/rateLimit.ts           — حماية من brute-force على login/register/checkout
src/i18n/                      — إعداد next-intl (اللغات، التنقل)
src/proxy.ts                   — كشف اللغة وإعادة التوجيه (اسم middleware الجديد في Next 16)
messages/{fr,en,ar}.json       — كل نصوص الواجهة بثلاث لغات
src/app/[locale]/product/[slug]/ — صفحة المنتج (القصة + المخزون المتبقي + المقاسات)
src/app/[locale]/checkout/     — الدفع: عنوان التوصيل + استبدال النقاط + تأكيد الطلب
src/app/[locale]/account/      — رصيد النقاط + سجل الطلبات + سجل النقاط
src/app/admin/                 — لوحة الإدارة (غير مترجمة عمداً — أداة داخلية بالفرنسية)
```

## تعدد اللغات (i18n)

- ثلاث لغات: الفرنسية (الافتراضية)، الإنجليزية، والعربية (RTL كامل عبر
  `dir="rtl"` وclasses منطقية في Tailwind مثل `end-`/`text-end`).
- كل رابط يحمل بادئة اللغة دائماً (`/fr/...`, `/en/...`, `/ar/...`) —
  الجذر `/` يعيد التوجيه حسب لغة المتصفح.
- محتوى المنتجات (الاسم، الشعار، القصة) مخزّن في قاعدة البيانات كـ jsonb
  بالشكل `{fr, en, ar}` مع fallback تلقائي للفرنسية إن كانت ترجمة ناقصة.
- اسم المنتج في الطلب (`orderItems.productName`) يُثبَّت بلغة الشراء وقتها —
  مثل إيصال ورقي، لا يتغير إن غيّر الزبون لغة الموقع لاحقاً.
- الأسعار كلها بالدرهم المغربي في كل اللغات (لا تحويل عملات) — فقط التنسيق
  واللاحقة يتغيران (`DH` / `MAD` / `د.م.`)، والأرقام تبقى لاتينية حتى بالعربية.
- `/admin` و`/api` خارج نظام الترجمة عمداً.

## القرارات التقنية المهمة (ولماذا)

- **PostgreSQL بدل SQLite.** النسخة الأولى من هذا المشروع استخدمت SQLite لأنها
  تعمل فوراً بدون أي خدمة خارجية. لكن أي استضافة مجانية شائعة (راجع
  DEPLOYMENT.md) لا تملك قرصاً دائماً — فأي ملف SQLite يُفقد عند كل إعادة
  تشغيل. رُحّل المشروع بالكامل إلى Postgres حقيقي (`drizzle-orm/node-postgres`)
  ليكون قابلاً للاستضافة فعلياً.
- **لا Stripe في مسار الدفع الفعلي.** Stripe لا يدعم حسابات تجار مسجلين
  بالمغرب. الدفع عند الاستلام (COD) هو الطريقة الوحيدة المفعّلة، لكن الكود
  مبني بطبقة فصل (`src/lib/payments.ts` + حقول `paymentMethod`/`paymentReference`
  عامة في جدول `orders`) لإضافة CMI أو PayZone أو ChariBaaS لاحقاً بدون
  إعادة هيكلة. التفاصيل الكاملة في **PAYMENTS.md**.
- **مصادقة يدوية بدل NextAuth.** لتفادي مخاطر توافق نسخة قد تتغير، وللتحكم
  الكامل في شكل الجلسة. الكوكيز httpOnly + secure في الإنتاج + SameSite=Lax.
- **معاملة واحدة (transaction) للشراء كاملاً.** التحقق من المخزون، خصمه،
  خصم/إضافة النقاط، كل هذا يحدث معاً أو لا يحدث شيء منه إطلاقاً — لا يوجد
  احتمال لبيع نفس القطعة الأخيرة مرتين. (معاملات Postgres هنا غير متزامنة
  async/await بالكامل — إن عدّلت `src/lib/orders.ts`، تأكد من استخدام `await`
  مع كل استعلام داخلها. نسيان `await` واحد هنا لا يُظهره فحص TypeScript، واكتُشف
  فقط عبر اختبار فعلي أثناء بناء هذا المشروع.)
- **Next.js 16 + Tailwind v4.** أحدث من فترة تدريب Claude، لذلك تحقق كل جزء من
  الكود مباشرة من التوثيق المرفق داخل `node_modules/next/dist/docs` وقت
  البناء، وليس من الذاكرة.

## قبل الإطلاق الفعلي — قائمة تحقق

0. **الترحيل الجديد (مهم لقاعدة بيانات موجودة).** الترحيل
   `drizzle/0001_cool_viper.sql` يحوّل نصوص المنتجات إلى jsonb متعدد اللغات
   ويضيف قيود CHECK على المخزون. شغّل `npm run db:migrate` ثم
   `npm run db:seed` (لتعبئة الترجمات EN/AR الحقيقية بدل نسخ النص الفرنسي).
   جرّبه على فرع Neon تجريبي قبل قاعدة الإنتاج.
1. **الاسم والشعار.** اقرأ BRANDING.md.
2. **الاستضافة.** اقرأ DEPLOYMENT.md — Vercel المجاني يمنع الاستخدام التجاري.
3. **بوابة الدفع.** اقرأ PAYMENTS.md.
4. **`JWT_SECRET`.** غيّره لقيمة عشوائية طويلة حقيقية قبل النشر
   (`openssl rand -base64 32`). لا تستخدم القيمة التجريبية في `.env.example`.
5. **صور المنتجات.** الصور الحالية في `public/products` هي نفس الصور التي
   رفعتها للمحادثة. استبدلها بتصوير منتج نهائي عند التوفر.
6. **المحتوى.** قصص المنتجات في `src/db/seed.ts` نص توضيحي بأسلوب العلامة —
   راجعها واستبدلها بالسرد الحقيقي الذي تريده لكل قطعة.

## ما هو غير مكتمل عمداً (نطاق واضح، لا خداع)

- لوحة إدارة بسيطة متوفرة على `/admin` (محميّة بكلمة سر عبر متغير البيئة
  `ADMIN_PASSWORD`): عرض الطلبات، تأكيدها/تسليمها/إلغاؤها، رقم المبيعات،
  والمخزون المتبقي. إضافة/تعديل المنتجات نفسها لا تزال عبر `seed.ts` أو
  `npm run db:studio`.
- لا بوابة دفع ببطاقة بنكية مفعّلة (انظر PAYMENTS.md).
- لا استرجاع لكلمة السر المنسية (لا يوجد إرسال بريد إلكتروني مُعد بعد).
- لا مبدّل لغة مرئي في الواجهة بعد — تغيير اللغة يتم بتعديل بادئة الرابط
  يدوياً (`/fr` ↔ `/en` ↔ `/ar`). إضافة زر تبديل في الـ Header خطوة صغيرة
  تالية منطقية.
- rate limiting يعمل في ذاكرة العملية الواحدة فقط (best-effort) — كافٍ لردع
  السكربتات البسيطة، لكنه ليس ضمانة موزّعة عبر عدة خوادم serverless.
